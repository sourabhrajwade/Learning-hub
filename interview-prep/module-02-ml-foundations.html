L1 Machine Learning Foundations (Learn First)

Before LLMs and agents, understand the fundamentals they're built on. This section is a learning layer — not interview Q&A — so the rest of the doc makes sense even if your background is software engineering more than ML.

What is Machine Learning?

Traditional programming: you write rules, the computer applies them.
Machine learning: you give the computer examples, and it figures out the rules. The "model" is just a mathematical function whose parameters were tuned by looking at data.

Example: instead of writing "if email contains 'free money' then spam", you show the model 100,000 emails labeled spam/not-spam and it learns the patterns — including ones you'd never think of.

The Three Types of ML

TypeWhat you give itWhat it learnsExample
Supervised Inputs + correct answers (labels) To map inputs → outputs Spam detection, credit scoring, image classification
Unsupervised Inputs only, no labels Hidden structure / groups Customer segmentation, anomaly detection
Reinforcement An environment with rewards Best actions to maximize reward Game-playing AI, robotics, RLHF for LLMs

LLMs are mostly supervised (predict the next token given previous tokens) with a layer of reinforcement learning on top (RLHF — humans rate responses, model learns to prefer good ones).

Classification vs Regression vs Generation

  • Classification — output is a category. "Is this transaction fraud? yes/no." "Which of 10 product categories?" Output: a probability for each class.
  • Regression — output is a number. "What's the house price?" "What's the credit risk score?" Output: a continuous value.
  • Generation — output is structured content (text, image, audio). LLMs are generative classifiers at their core: at each step they classify "what's the next token?" over a vocabulary of ~100k options.

How a Model Actually Learns (The Training Loop)

  1. Initialize the model with random parameters (weights).
  2. Forward pass: feed an input, get a prediction.
  3. Loss: compare prediction to the correct answer; compute how wrong it was (a single number).
  4. Backward pass (backprop): compute how each weight contributed to the error.
  5. Update: nudge each weight slightly in the direction that reduces the loss (gradient descent).
  6. Repeat on millions of examples until the loss stops dropping.

That's it. Everything from a logistic regression to GPT-4 follows this same loop — they differ in architecture (the shape of the function) and scale (how much data and how many parameters).

Critical Vocabulary — Memorize These

TermPlain English
FeaturesThe input variables (columns in a spreadsheet). For text, the features are the tokens.
Labels / TargetsThe correct answers you're trying to predict.
Parameters / WeightsThe numbers inside the model that get tuned during training. GPT-4 has ~1.8 trillion.
HyperparametersKnobs you set before training (learning rate, batch size). Not learned.
Loss functionA formula measuring how wrong the model is. Training = minimizing loss.
Gradient descentThe algorithm that updates weights to reduce loss. "Roll downhill on the loss landscape."
EpochOne full pass over the training data.
BatchA small group of examples processed together. GPUs work on batches, not single rows.
OverfittingModel memorized training data but fails on new data. Like a student who memorized answers without understanding.
UnderfittingModel is too simple to capture the pattern. Trains poorly AND tests poorly.
Train / Validation / Test splitTrain on one set, tune on a second, judge final quality on a third (untouched until the end).
InferenceUsing a trained model to make predictions. (Training is one-time; inference is continuous in production.)

How We Measure "Good"

  • Accuracy — % correct. Misleading when classes are imbalanced (99% non-fraud → predicting "never fraud" gets 99% accuracy and is useless).
  • Precision — of the items I flagged as positive, how many actually were? (Avoid false alarms.)
  • Recall — of the actual positives, how many did I catch? (Don't miss real cases.)
  • F1 — harmonic mean of precision and recall. One number that balances both.
  • AUC-ROC — how well the model separates classes across all decision thresholds. 0.5 = random, 1.0 = perfect.
  • RMSE / MAE — for regression — how far off your numerical predictions are on average.

Banking intuition: in fraud detection, recall matters more than accuracy — missing real fraud is far more expensive than a few false alarms. In credit decisioning, precision on the "approve" class matters more — a wrong approval becomes a default loss.

From Classical ML to Deep Learning to LLMs

  • Classical ML (logistic regression, decision trees, gradient boosting) — works on structured data (tables). Fast to train, easy to interpret. Still the right answer for most tabular problems in banking (XGBoost is the go-to).
  • Deep learning — neural networks with many layers. Works directly on unstructured data (text, images, audio) by learning their own features. Needs lots of data and GPUs.
  • Transformers (2017, "Attention is All You Need") — the architecture that broke open language. Uses self-attention to let every token "look at" every other token. Scales spectacularly with data and compute.
  • Large Language Models — transformers trained on a huge fraction of the internet to predict the next token. The "magic" is what emerges from scale: reasoning, code, translation, none of it explicitly programmed.
Key insight for the role: not every problem needs an LLM. If the input is a table and the output is a number or a category, classical ML is faster, cheaper, and more interpretable. LLMs are for language-shaped problems — where the input or output is unstructured text.
