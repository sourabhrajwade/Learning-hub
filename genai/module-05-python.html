1 Python for Application & AI Development

Expect questions on idiomatic Python, async, packaging, testing, and how you'd structure an AI service.

Q1 Walk me through how you'd structure a production-grade Python service that wraps an LLM API.

Layers: (1) FastAPI/Flask entry layer with Pydantic schemas for validation, (2) a service layer with provider-agnostic LLM client interface, (3) a prompt/template module versioned in code, (4) an evaluation + telemetry layer (OpenTelemetry, structured JSON logs, token/cost metrics), (5) async I/O for concurrent calls, (6) tests at unit, contract, and eval levels. Packaging: pyproject.toml, uv or poetry, pinned deps. Config: 12-factor with pydantic-settings. Resilience: retries with exponential backoff + jitter, circuit breaker, request timeouts.

Q2 Sync vs async in Python — when does it matter for AI workloads?

LLM calls are I/O-bound and can take seconds — async lets a single worker handle hundreds of concurrent requests. Use asyncio + httpx.AsyncClient for outbound calls. CPU-bound work (embeddings on local models, tokenization at scale) should run in a process pool, not the event loop. Don't mix blocking and async — that's the classic foot-gun.

Q3 Code: write a streaming LLM endpoint.
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
from anthropic import AsyncAnthropic

app = FastAPI()
client = AsyncAnthropic()

class Req(BaseModel):
    prompt: str

@app.post("/chat")
async def chat(req: Req):
    async def gen():
        async with client.messages.stream(
            model="claude-opus-4-7",
            max_tokens=1024,
            messages=[{"role": "user", "content": req.prompt}],
        ) as stream:
            async for text in stream.text_stream:
                yield text
    return StreamingResponse(gen(), media_type="text/event-stream")
Q4 How do you test code that calls an LLM?

Three layers: (1) Unit tests with the LLM mocked — verify prompt construction, parsing, error handling. (2) Contract tests against a recorded fixture (VCR-style) — catches API shape regressions cheaply. (3) Evaluation tests that hit the real model on a curated dataset and assert on rubrics (faithfulness, format, refusal). Run evals in CI gated on a budget; treat regressions like test failures.
